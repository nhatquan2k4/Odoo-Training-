# Version Control và Git

## Mục tiêu

Hiểu Version Control là gì và vì sao cần sử dụng Git trong quá trình phát triển phần mềm.

## Version Control là gì?

Version Control là hệ thống giúp theo dõi và quản lý sự thay đổi của file hoặc toàn bộ dự án theo thời gian.

Khi sử dụng Version Control, ta có thể:

- Xem lại lịch sử thay đổi của dự án.
- Biết ai đã thay đổi nội dung nào và vào thời điểm nào.
- So sánh sự khác nhau giữa các phiên bản.
- Khôi phục lại phiên bản cũ khi cần.

## Vì sao cần Version Control?

Trong quá trình làm việc, mã nguồn thường xuyên được chỉnh sửa, thêm mới hoặc xóa bỏ. Nếu không có công cụ quản lý phiên bản, rất khó để kiểm soát thay đổi và khôi phục khi xảy ra lỗi.

Version Control giúp dự án an toàn hơn, dễ theo dõi hơn và hỗ trợ làm việc nhóm hiệu quả hơn.

## Vì sao cần Git?

Git là một hệ thống Version Control phổ biến hiện nay. Git giúp lập trình viên quản lý lịch sử mã nguồn, làm việc độc lập trên máy cá nhân và đồng bộ thay đổi với các thành viên khác trong nhóm.

Git cần thiết vì:

- Lưu lại toàn bộ lịch sử thay đổi của dự án.
- Cho phép quay lại phiên bản trước nếu code bị lỗi.
- Hỗ trợ nhiều người cùng làm việc trên một dự án.
- Có thể làm việc offline trên máy cá nhân.
- Dễ kết hợp với các nền tảng như GitHub, GitLab hoặc Bitbucket.

## Kết luận

Version Control giúp quản lý thay đổi của dự án. Git là công cụ Version Control giúp việc phát triển phần mềm trở nên an toàn, rõ ràng và thuận tiện hơn, đặc biệt khi làm việc nhóm.
